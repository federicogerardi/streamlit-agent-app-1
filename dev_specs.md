Architettura del Flusso Dati (Logical Blueprint)L'applicazione segue un modello di esecuzione lineare ma ottimizzato tramite frammenti per gestire lo streaming dei dati dall'API di OpenRouter senza invalidare l'intera interfaccia utente.Ciclo di Vita del Dato:Fase di Autenticazione: All'avvio, l'app verifica st.user.is_logged_in. Se False, viene invocato st.login() forzando il redirect verso l'OIDC provider (es. Google o Microsoft).Inizializzazione Risorse: Il client di connessione a OpenRouter viene istanziato una sola volta e mantenuto in memoria tramite @st.cache_resource.Input Utente: L'utente interagisce con st.chat_input. Questo triggera un aggiornamento del st.session_state["messages"].Generazione & Streaming: Viene invocata una funzione decorata con @st.fragment che gestisce la chiamata API a OpenRouter e il rendering immediato dei token tramite st.write_stream.Persistenza Locale: A generazione completata, la risposta finale viene salvata permanentemente nello stato della sessione per mantenere lo storico durante i successivi rerun.2. 4. Guida all'Implementazione per SviluppatoriIntegrazione OpenRouter (Pattern Consigliato):OpenRouter è compatibile con l'SDK OpenAI. Utilizzare il client ufficiale configurando l'header HTTP-Referer e X-Title come richiesto dai termini di servizio di OpenRouter.Gestione Errori:Auth Error: Se l'autenticazione fallisce, utilizzare st.error con un pulsante di retry.API Timeout: Gestire i timeout delle API di OpenRouter avvisando l'utente tramite un messaggio informativo (st.info) senza interrompere l'app.Testabilità (st.testing.v1):Gli sviluppatori devono implementare test automatici per:Verificare che st.chat_input sia disabilitato se l'utente non è loggato.Simulare la ricezione di un errore 401 dall'API e verificare che venga mostrato l'errore corretto nella UI.5. Pillola di Mentoring dell'Architetto: "Streaming UI Isolation"In Streamlit, ogni rerun del backend ridisegna l'intera pagina. In un'app di chat, questo può risultare pesante.Best Practice: Inserite la logica di visualizzazione dei messaggi e il st.write_stream all'interno di un componente isolato con @st.fragment.Questo permette di:Mantenere lo stato dello scroll della finestra di chat stabile.Evitare che componenti "pesanti" nella sidebar (come liste di modelli caricate via rete) vengano ricalcolati ogni volta che un nuovo chunk di testo arriva dall'LLM.Il risultato è un'app che si percepisce come una Single Page Application (SPA) nativa, pur essendo scritta interamente in Python.
